# ====================================================
#   VCP 型態（波動收縮型態）買入與賣出策略
#   適用於：量化交易系統訊號引擎 / 策略擴充
# ====================================================

【VCP 型態核心定義】
1. 趨勢先行：股價處於明確的上升趨勢中（通常位於長線 50日、100日 或 200日 SMA 之上）。
2. 波動收縮（Contractions）：股價經歷 2 到 4 次的價格拉回（V1 -> V2 -> V3...），且每次拉回的幅度（高點到低點的百分比）呈現遞減趨勢（例如：25% -> 15% -> 8% -> 3%）。
3. 價格停頓區（Cheat/Pivot）：在最後一次收縮的尾端，價格波動極小，形成一個狹窄的盤整區間。
4. 量能配合：在每次收縮的底部以及 pivot 區，成交量必須顯著萎縮；突破時則需要爆量。


# ----------------------------------------------------
#  一、 VCP 買入點位（Entry Points）
# ----------------------------------------------------

1. 標準樞軸點突破買入（Pivot Breakout）- 主要買點
   * 條件：
     - 股價突破最後一次收縮（波動幅度已萎縮至 1% ~ 5% 內）的最高點（即 Pivot 阻力線）。
     - 同時伴隨【量能確認】，當日成交量必須大於前 20 日平均成交量的 1.5 倍至 2 倍以上。
   * 動作：當突破發生時，立即觸發 BUY 訊號。

2. 狹幅盤整區低量預備買入（Cheat Entry）- 進階早鳥買點
   * 條件：
     - 在經歷至少 3 次波動收縮後，股價在 Pivot 區連續數日呈現「極窄幅橫盤」（High-Low 區間極小）。
     - 成交量萎縮至近期窒息量（低於 20 日均量的 50% 以下）。
   * 動作：在盤整區下緣或突破盤整當日提早建立基本倉位（通常為預定資金的一半），以獲取價格優勢。


# ----------------------------------------------------
#  二、 VCP 賣出與風控點位（Exit & Risk Control Points）
# ----------------------------------------------------

1. 跌破樞軸點停損（Stop Loss）
   * 條件：
     - 買入後，股價並未如預期噴出，反而跌回 Pivot 區間。
     - 跌破 Pivot 區間底部的支撐線，或觸及 3% ~ 5% 的硬停損比例（短線風控）。
   * 動作：立即觸發 SELL 訊號，執行停損，避免跌入深谷。

2. 跌破關鍵移動平均線（Trend Exit）
   * 條件：
     - 股價突破後未能延續漲勢，且跌破了短期的 10 EMA 或 20 EMA。
     - 長線持有者則以跌破 50 日 SMA 作為部分止損或全出訊號。
   * 動作：觸發 SELL 訊號，保護本金。

3. 爆量高檔滯漲 / 超買止盈（Take Profit）
   * 條件：
     - 突破後股價連續急漲，短期內乖離率過高。
     - RSI 進入 > 70 或 > 80 的嚴重超買區。
   * 條件二（高檔滯漲）：出現爆量但股價卻收黑K（或長上影線），代表主力出貨。
   * 動作：依據 1:2 或 1:3 的盈虧比分批獲利了結（SELL）。


# ----------------------------------------------------
#  三、 系統整合建議 (與 quant_system 結合)
# ----------------------------------------------------
* 自動交易清單篩選：
  - 可以利用 XS 腳本或後台 `signal_engine.py` 先篩選出符合「收縮幅度遞減」且「站於均線之上」的股票加入 Telegram 清單（/add）。
* 監控與執行：
  - `executor.py` 每分鐘檢查清單時，若發現價格突破設定的 Pivot 價位且量能達標，則執行下單。
  - 當風控模組偵測到黑天鵝事件時，若符合 VCP 停損條件，須判斷是否屬於系統性系統風險再決定是否賣出。
